# UrbanNest Estate

UrbanNest Estate - учебный PHP + MySQL проект для подбора недвижимости. Приложение работает через XAMPP и показывает полный путь пользователя: от просмотра каталога до отправки заявки на объект.

Проект подготовлен студентами Splavski Maxim и Metelska Daniela.

## Идея проекта

UrbanNest Estate задуман как спокойный и деловой каталог объектов недвижимости. В нем можно просматривать квартиры, дома, коммерческие помещения, участки и новостройки, а затем оставлять заявку на просмотр.

## Что реализовано

- главная страница с крупным блоком поиска;
- каталог объектов недвижимости;
- поиск по названию, городу и району;
- фильтры по типу объекта, сделке, комнатам, статусу, цене и площади;
- сортировка по цене, площади и дате добавления;
- отдельная страница объекта;
- регистрация, вход и выход пользователя;
- личный кабинет со списком заявок;
- форма заявки на просмотр;
- административный dashboard;
- управление объектами недвижимости;
- управление типами объектов;
- обработка заявок;
- управление пользователями и создание администраторов;
- локальное хранение и автоматическая обработка изображений.

## База данных

База данных называется `urbannest_estate`.

В проекте используются четыре основные таблицы:

- `users` - аккаунты пользователей, телефоны, email, роли и хеши паролей;
- `property_types` - справочник типов недвижимости;
- `properties` - сами объекты недвижимости с адресом, ценой, площадью, статусом и фотографией;
- `property_requests` - заявки пользователей на просмотр объектов.

Связи устроены просто: объект относится к типу недвижимости, а заявка связана с пользователем и выбранным объектом.

## Запуск

1. Убедиться, что папка проекта находится внутри `htdocs`.
2. Запустить Apache и MySQL в XAMPP.
3. Открыть phpMyAdmin.
4. Импортировать файл `database/database.sql`.
5. Проверить данные подключения в `app/config/config.php`.
6. Открыть проект:

```text
http://localhost/projects/usm/li/Splavski%20Maxim%20-%20Metelska%20Daniela/public/
```

## Основные сценарии

Гость открывает главную страницу, вводит запрос или переходит в каталог. В каталоге он выбирает тип недвижимости, формат сделки, количество комнат, цену и площадь.

После регистрации пользователь может открыть страницу объекта и отправить заявку на просмотр. Все созданные заявки отображаются в личном кабинете вместе со статусом обработки.

Администратор входит в отдельный раздел, добавляет или редактирует объекты, меняет типы недвижимости, просматривает заявки и управляет ролями пользователей.

## Изображения

Стартовые фотографии объектов находятся в:

```text
public/assets/img/properties/
```

Фотографии, которые администратор загружает через форму объекта, сохраняются в:

```text
public/uploads/properties/
```

В базе данных хранится только относительный путь к изображению, например:

```text
assets/img/properties/apartment-central-residence.jpg
uploads/properties/20260506191000-a1b2c3d4.jpg
```

Поддерживаются исходные форматы `JPG`, `PNG` и `WEBP`. Файл должен быть минимум `900x900` пикселей. После загрузки приложение обрезает изображение по центру, приводит его к квадрату `900x900` и сохраняет результат как `JPG`.

## Безопасность

- пароли сохраняются только через `password_hash()`;
- вход проверяется через `password_verify()`;
- приватные страницы требуют активной сессии;
- административные страницы требуют роль `admin`;
- SQL-запросы выполняются через подготовленные PDO-запросы;
- вывод данных проходит через функцию экранирования `e()`;
- изображения проверяются по формату, размеру файла и разрешению.

## Отчет

LaTeX-файлы находятся в папке `report/`.

Для компиляции используется XeLaTeX. Шаблон `common.tex` подключается из той же папки, поэтому отчет можно компилировать из директории `report`.
