# UrbanNest Estate

UrbanNest Estate - традиционное PHP + MySQL веб-приложение для просмотра и подбора современной недвижимости.

Работа выполнена двумя студентами: Splavski Maxim и Metelska Daniela.

## Цель работы

Создать завершенный MVP веб-приложения средней сложности с публичным каталогом недвижимости, регистрацией пользователей, заявками на просмотр, личным кабинетом, ролями и административной панелью.

## Используемые технологии

- HTML5, CSS3, JavaScript
- PHP 8+
- MySQL
- Apache, XAMPP, phpMyAdmin
- PDO prepared statements
- PHP sessions
- MVC-like modular architecture
- XeLaTeX для отчета

## Функциональные возможности

- регистрация, вход и выход пользователей;
- безопасное хранение паролей через `password_hash()`;
- публичный каталог объектов недвижимости;
- поиск по названию, городу и району;
- фильтр по типу объекта, типу сделки, комнатам, статусу, цене и площади;
- сортировка по цене, площади и дате добавления;
- детальная страница объекта;
- форма заявки на просмотр с клиентской и серверной валидацией;
- личный кабинет пользователя со списком заявок;
- административная панель со статистикой;
- CRUD для объектов недвижимости;
- CRUD для типов недвижимости;
- управление заявками и пользователями;
- создание новых администраторов;
- адаптивный темный деловой интерфейс.

## Архитектура проекта

```text
app/
  config/        настройки приложения и базы данных
  controllers/   обработка запросов
  core/          подключение к БД и вспомогательные функции
  models/        работа с таблицами MySQL
  views/         HTML-шаблоны страниц
database/
  database.sql   структура и тестовые данные
public/
  index.php      единая точка входа
  assets/        CSS, JavaScript и локальные изображения недвижимости
  uploads/       изображения, загруженные администратором
report/
  report.tex     отчет по проектной работе
common.tex       общий LaTeX-шаблон титульного листа
```

## Структура базы данных

- `users` - пользователи, телефоны, email, хеш пароля, роль.
- `property_types` - типы недвижимости: Apartment, House, Commercial, Land, New Building.
- `properties` - объекты недвижимости с характеристиками, статусом, ценой и путем к изображению.
- `property_requests` - заявки пользователей на просмотр объектов.

## Запуск через XAMPP

1. Скопировать папку проекта в `htdocs`.
2. Запустить Apache и MySQL в XAMPP.
3. Открыть phpMyAdmin.
4. Импортировать файл `database/database.sql`.
5. Проверить настройки подключения в `app/config/config.php`.
6. Открыть проект в браузере:

```text
http://localhost/projects/usm/li/Splavski%20Maxim%20-%20Metelska%20Daniela/public/
```

## Примеры использования

1. Гость открывает главную страницу и выполняет быстрый поиск.
2. Гость фильтрует каталог по городу, типу сделки, комнатам и цене.
3. Пользователь регистрируется и входит в систему.
4. Пользователь выбирает объект и отправляет заявку на просмотр.
5. Администратор входит в панель управления.
6. Администратор меняет статус заявки, добавляет объект или редактирует тип недвижимости.

## Меры безопасности

- доступ к личному кабинету только после входа;
- доступ к админ-панели только для роли `admin`;
- пароли хранятся только в хешированном виде;
- SQL-запросы выполняются через PDO prepared statements;
- вывод пользовательских данных экранируется функцией `e()`;
- формы проверяются на стороне клиента и сервера;
- ошибки базы данных не выводятся пользователю в техническом виде.

## Хранение изображений

Изображения объектов хранятся локально. Для тестовых данных используются файлы из:

```text
public/assets/img/properties/
```

Если администратор загружает новое изображение через форму объекта, файл сохраняется в:

```text
public/uploads/properties/
```

В базе данных в поле `properties.image` хранится не сама картинка, а относительный путь:

```text
assets/img/properties/apartment-central-residence.jpg
uploads/properties/20260506191000-a1b2c3d4.jpg
```

Поддерживаются исходные форматы `JPG`, `PNG` и `WEBP`. Загружаемое изображение должно быть не меньше `900x900` пикселей. После загрузки приложение автоматически делает центральную квадратную обрезку, уменьшает результат до `900x900` и сохраняет итоговый файл как `JPG`.

Например, вертикальная фотография `1200x1600` будет обрезана по центру до квадрата и сохранена как готовая карточная фотография `900x900`.

## Контрольные вопросы

1. Серверная часть веб-приложения обрабатывает запросы, работает с базой данных и формирует HTML-страницы.
2. База данных нужна для постоянного хранения пользователей, объектов недвижимости, типов объектов и заявок.
3. Аутентификация - это проверка личности пользователя по email и паролю.
4. Авторизация определяет, какие действия доступны пользователю после входа.
5. Пароли нельзя хранить открыто, потому что при утечке базы их сразу смогут использовать злоумышленники.
6. PHP-сессии сохраняют состояние входа пользователя между запросами.
7. SQL-инъекция - внедрение вредоносного SQL-кода через пользовательский ввод.
8. Подготовленные запросы отделяют SQL-код от данных и защищают от SQL-инъекций.
9. Валидация данных проверяет корректность введенной информации перед сохранением.
10. Административная панель нужна для управления данными приложения.
11. Роль пользователя определяет уровень доступа, например `user` или `admin`.
12. Адаптивная верстка важна, чтобы сайтом было удобно пользоваться на телефоне и компьютере.
13. MVC разделяет модели данных, обработчики запросов и представления.
14. Модульная архитектура упрощает поддержку, проверку и расширение проекта.
